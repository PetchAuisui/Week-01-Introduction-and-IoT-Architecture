# Prompt หมวด A — ข้อที่ 1: Introduction to IoT
## Question
```
คุณคือ TA วิชา 03376134 APPLICATIONS OF INTERNET OF THINGS  
หน้าที่ของคุณคือทำการ audit คำตอบของฉันตามเกณฑ์ต่อไปนี้:

คำถาม:  
“อธิบายว่า Internet of Things (IoT) คืออะไร และยกตัวอย่างระบบ IoT ที่พบในชีวิตประจำวัน พร้อมอธิบายว่าทำไมระบบนั้นจึงถือเป็น IoT”

กรุณาตรวจคำตอบของฉันตาม rubric ต่อไปนี้:

1) ความถูกต้องของเนื้อหา (40%)  
   - อธิบายความหมายของ IoT ถูกต้องหรือไม่  
   - ตัวอย่างที่ยกมาเป็น IoT จริงหรือไม่  
   - อธิบายเหตุผลว่าทำไมระบบนั้นเป็น IoT ได้ถูกต้องหรือไม่  

2) ความครบถ้วนของคำอธิบาย (30%)  
   - มีทั้งนิยาม + ตัวอย่าง + เหตุผลประกอบ  
   - มีความเข้าใจเชิงระบบหรือไม่  

3) ความเป็นภาษามนุษย์ (Human Authenticity) (30%)  
   - ตรวจว่าคำตอบของฉันมีลักษณะภาษาที่ AI ชอบใช้หรือไม่ เช่น  
     • การใช้ ":" มากเกินไป  
     • การใช้ emoji  
     • การใช้ bullet point แบบ textbook  
     • การใช้ประโยคสูตรสำเร็จ เช่น “ดังนั้น”, “อย่างไรก็ตาม”, “กล่าวคือ”  
   - ตรวจว่ามีความเป็นธรรมชาติของมนุษย์หรือไม่  
   - ตรวจว่ามีการอธิบายด้วยภาษาของตัวเองหรือไม่  
   - ตรวจว่ามีความผิดพลาดเล็กน้อยแบบมนุษย์หรือไม่ (เป็นเรื่องปกติ)

ให้คะแนนทั้งสามหมวด พร้อมคำอธิบายเหตุผลอย่างละเอียด  
และสรุปคะแนนรวม 100 คะแนน

สุดท้าย ให้ข้อเสนอแนะว่าฉันควรปรับปรุงตรงไหนเพื่อให้ได้คะแนนสูงขึ้น
```

## Answer
```
IoT หรือ Internet Of Things คือ การเชื่อมต่อโลกระหว่าง โลก physical กับ digital โดยทำให้ข้อมูลจริงนำเข้าสู่โลก digital และทำให้คำสั่งจากโลก digital สามารถส่งกลับมาควบคุมในโลกจริงได้ ทำให้ระบบสามารถรับรู้สถานการณ์จริง นำมาวิเคราะห์ และตอบสนองอัตโนมัติ โดยอุปกรณ์ไม่สามารถเชื่อมต่อกับระบบ digital โดยตรงได้เราเลยจะเชื่อมต่อ physical ผ่าน พวก

- sensor ทำหน้าที่ตรวจวัดปรากฏการณ์ทางกายภาพ เช่น อุณหภูมิ ความชื้น แรงดัน หรือ การเคลื่อนไหว
- actuator ทำหน้าแปลงสัญญาณรูปแบบ digital กลับไปให้แสดงผลบนโลกจริง
- data converter  ทำหน้าที่แปลงสัญญาณจากการวัดให้เป็นข้อมูล digital ที่ระบบนำไปประมวลผลได้

สรุป IoT คือ ระบบที่ทำให้โลกของความเป็นจริง สามารถ รายงาน หรือ รับคำสั่ง จาก โลกดิจิตอลได้ผ่านตัว sensor actuator และ network 

ตัวอย่าง IoT ใน ชีวิต ประจำวัน

- smartwatch ใช้นับก้าวเดิน วัดอัตราการเต้นของหัวใจ ติดตามการนอนหลับ ข้อมูลจะซิงค์กับโทรศัพท์เพื่อดูข้อมูลได้
- ระบบติดตามพัสดุ ใช้ GPS กับ sensor ที่ติดกับรถขนพัสดุ ทำให้รู้ตำแหน่งของพัสดุว่าอยู่ตรงไหน
- ตู้เต่าบิน มีการแจ้งเตือนตอนวัตถุดิบใกล้หมดตู้จะส่งแจ้งเตือนไปยังพนักงาน
- ม่านกั้นรถไฟฟ้า ใช้ Sensor เเล้วส่งข้อมูลไปเช็คกับระบบหลังบ้านทันทีแล้วเปิดประตูให้เรา
```

# Prompt หมวด B — ข้อที่ 2: Application Layer Technology
## Question
```
คุณคือ TA วิชา  03376134 APPLICATIONS OF INTERNET OF THINGS  
หน้าที่ของคุณคือทำการ audit คำตอบของฉันตามเกณฑ์ต่อไปนี้:

คำถาม:  
“อธิบายโปรโตคอลในชั้น Application Layer ที่ใช้ในระบบ IoT เช่น MQTT, AMQP, XMPP  
พร้อมอธิบายความแตกต่างของแต่ละโปรโตคอล และยกตัวอย่างสถานการณ์ที่เหมาะสมกับแต่ละโปรโตคอล”
```

## Answer
```
Application Layer มีความสำคัญเนื่องจาก อุปกรณ์ IoT มีข้อจำกัดเรื่อง พลังงาน หน่วยความจำ และความสเถียรของเครือข่าย ที่ละอุปกรณ์แตกต่างกันไป

- MQTT (Message Queuing Telemetry Transport) ทำงานอยู่บนสถาปัตยกรรมแบบ Publish หรือ Subscribe โดยมี Message Broker เป็นตัวกลางในการจัดสรรข้อมูล อุปกรณ์ฝั่งส่ง จะส่งข้อมูลระบุหัวข้อไปที่ Broker ตัว Broker จะไม่เก็บข้อมูลไว้ แต่จะดูทันทีว่ามีใครลงทะเบียนรอรับหัวข้อนี้อยู่บ้างไหม เเล้วก็ส่งต่อไปข้อมูลไปยังอุปกรณ์ทุกตัวที่ Subscribe นั้นไว้จุดเด่น เป็น protocol ที่ Lightweight หรือ น้ำหนักเบามาก ส่วนหัวของ packet หรือ Header มีขนาดเริ่มต้นเพียง 2 Bytes เท่านั้น ออกมาเพื่ออุปกรณ์ขนาดเล็กที่มีทรัพยากรจำกัด และทำงานได้ดีบนเครือข่ายที่มีแบนด์วิดท์ต่ำหรือไม่เสถียร มีระบบ QoS (Quality of Service) 3 ระดับเพื่อควบคุมความน่าเชื่อถือในการส่งข้อมูล
- AMQP (Advanced message Queuing Protocol) ทำงานแบบ Publish หรือ Subscribe และ Point to point ผ่านกลไกที่ซับซ้อนขึ้นกว่า MQTT มี Exchange ทำหน้าที่ รับข้อความเเล้วใช้เงื่อนไขในการส่งต่อไปยัง Queues ต่างๆก่อนจะผู้ส่งถึงผู้รับจุดเด่น เน้นความน่าเชื่อถือและความปลอดภัยในระดับองค์กรรองรับการทำธุรกรรมข้อมูล เพื่อประกันว่าข้อความจะไม่สูญหายและเรียงลำดับถูกต้องแน่นอน ตัวโปรโตคอลเป็นแบบ Binary ที่มีความสามารถในการจัดการคิวที่ซับซ้อนแต่แลกมาด้วยการกินทรัพยากรและการประมวลผลที่สูงกว่า MQTT
- XMPP (Extensible Messaging and Presence Protocol) มีพื้นฐานมาจากภาษา XML ทำงานบนสถาปัตยกรรมแบบกระจายศูนย์ หรือ Decentralized Client-Server) คล้ายกับระบบ Email ที่ server แต่ละโดเมนสามารถคุยกันเองได้จุดเด่น มีจุดเเข็งที่สุดในเรื่องการจัดการ Presence การเช็กสถานะ ออนไลน์หรือออฟไลน์ ของอุปกรณ์แบบ Real-time โครสร้างเป็นข้อความ XML ทำให้มีความยืดหยุ่นสูง สามารถขยายโครงสร้างข้อมูลได้ง่าย แต่เนื่องจากใช้ XML ในการสื่อสาร จึงส่งผลให้ขนาดของแพ็กเกจข้อมูลมีขนาดใหญ่กว่า protocol แบบ Binary

สรุป
สถาปัตยกรรม
- MQTT  เป็นแบบPublish / Subscribe
- AMQP เป็นแบบ Publish / Subscribe และ Point-to-Point
- XMPP เป็นแบบ Decentralized Client-Server
รูปแบบข้อมูล
- Binary ไม่กำหนดรูปแบบอิสระ
- Binary มีโครงสร้างซับซ้อน ปลอดภัยสูง ข้อมูลไม่หาย
- XML  ข้อความตัวอักษร
ขนาดของ Header
- เล็กมาก เริ่มต้น 2 Bytes
- ปานกลาง-ใหญ่
- ใหญ่ เนื่องจากเป็น XML tags
การใช้พลังงานหรือทรัพยากร
- ต่ำมาก เหมาะกับอุปกรณ์เล็ก ประหยัดแบนด์วิดท์ กินไฟน้อย
- สูง เหมาะกับ Server/Gateway
- ปานกลาง-สูง ต้องใช้ XML Parser เช็คสถานะการเชื่อต่อได้ดี

สถานการณ์ที่เหมาะสมและเหตุผลประกอบ
- MQTT ระบบเกษตรกรรมอัจฉริยะ (Smart Agriculture)
การติดตั้ง sensor วัดความชื้นในดินและอุณหภูมิกลางทุ่งนาอันห่างไกล โดยใช้พลังงานจาก โซลาร์เซลล์ขนาดเล็ก และส่งข้อมูลผ่านเครือข่ายเซลลูลาร์ ข้อมูลมีขนาดเล็กและกินไฟต่ำช่วยประหยัดพลังงานต่ำทำให้อุปกรณ์ใช้ได้นานหลายปี
- AMQP ระบบจัดการหุ่นยนต์และคลังสินค้าในโรงงานอุตสาหกรรม (Smart Factory)
การเชื่อมต่อข้อมูลระหว่างสายพานการผลิต หุ่นยนต์ยกสินค้า และระบบบริหารทรัพยากรองค์กร หรือระบบการเงินของโรงงาน ในโรงงานข้อมูล ข้อมูลทุกชิ้นมีความสำคัญสูง หากมีข้อมูลหาย อาจทำให้เกิดความเสียหายได้
- XMPP ระบบควบคุมบ้านอัจฉริยะ (Smart Home Automation)
ระบบที่ผู้ใช้ควบคุมเครื่องใช้ไฟฟ้าภายในบ้าน เช่น เปิด-ปิดไฟ, ปรับอุณหภูมิแอร์, หรือดูสถานะกล้องวงจรปิดผ่านสมาร์ทโฟนแบบ Real-time ระบบ smarthome ต้องการการตอบสนองที่รวดเร็ว และต้องรู้สถานะปัจจุบัน ของอุปกรณ์ตลอดเวลาว่าหลุดจากการเชื่อมต่อหรือไม
```
